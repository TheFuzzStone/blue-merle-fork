# Apple, Inc. OUI prefixes (first 3 octets of MAC address).
#
# These are real, IEEE-registered Apple OUIs. Presenting a MAC from
# this list makes Mudi's radio identity look like a genuine Apple
# device (iPhone / iPad / MacBook / AirPods) to any DHCP server or
# WiFi tracker doing vendor lookup.
#
# Trade-offs (see LOCAL_CHANGES.md for the full note):
#   + Matches the iPhone/iPad hostname we advertise.
#   + Avoids the "locally-administered bit set" flag that WIDS
#     systems interpret as "randomized/spoofed MAC".
#   - Not RFC 7844-compliant (that RFC recommends U/L=1).
#   - Small chance of colliding with a real Apple device in the same
#     LAN. If that happens on your uplink, remove the offending OUI
#     from this list or fall back to blue-merle-newmac --uplink to
#     regenerate.
#
# Format: one OUI per line, lowercase hex, colon-separated, no
# trailing colon. Comments and blank lines are ignored.
#
# You may add/remove entries; changes take effect at next boot or
# after `service blue-merle reload`.
