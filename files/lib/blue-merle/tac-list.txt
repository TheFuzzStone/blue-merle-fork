# TAC (Type Allocation Code) list for IMEI generation.
#
# The first 8 digits of an IMEI identify the device model. Operators
# perform an automatic TAC-lookup at network registration. If the TAC
# says "Samsung Galaxy S22" but the device behaves like a Linux LTE
# router (DHCP fingerprint, Cat 6 capabilities, no voice), the operator
# raises a capability-mismatch flag.
#
# This list contains TACs allocated to LTE *modules* (not consumer
# smartphones) — devices that behave in the network exactly like the
# Mudi's Quectel EP06: data-only, LTE Cat 4-6, Linux-based. This makes
# the TAC-lookup consistent with the device's actual behaviour.
#
# Trade-off: the anonymity set is smaller (dozens of industrial gateway
# models, not millions of smartphones), but consistency at the TAC-lookup
# layer is more valuable than a large set with an obvious mismatch.
#
# If your threat model prefers "blend in with millions of phones" over
# "look like a consistent industrial gateway", replace this file with
# smartphone TACs (35xxxxxx range) — but be aware that operators with
# capability-checking will flag the mismatch.
#
# Format: one TAC per line, exactly 8 digits. Lines starting with '#'
# and blank lines are ignored. Invalid entries are silently skipped.
#
# You can edit this file in place; changes take effect at the next IMEI
# rotation (blue-merle CLI, toggle, or LuCI).
