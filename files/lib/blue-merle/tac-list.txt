# Module-mode TAC policy
#
# Module mode does NOT choose from this file. It preserves the first
# 8 digits of the IMEI currently reported by the physical modem.
# This avoids unsupported manufacturer/model claims and does not need
# licensed GSMA Device Database data.
#
# This file is retained only as documentation and as an optional place
# for a user to maintain their own verified TACs. To use an explicit
# list, set BLUE_MERLE_TAC_LIST to its path before invoking
# imei_generate.py.
#
# Do not add TACs without recording an authoritative source, dataset
# version/date, and the exact allocation record. TAC prefixes alone do
# not encode manufacturer or device class.
